# 血管卫士：生死时速 - H5游戏技术设计

## 一、页面尺寸与适配方案

### 设计基准

| 项目 | 规格 |
|------|------|
| 设计稿基准 | 375 x 812 px（iPhone标准竖屏比例） |
| 实际渲染 | 使用 `rem` + `viewport` 缩放，适配所有移动端屏幕 |
| PC端适配 | 页面居中显示，最大宽度 480px，两侧留白，保持竖屏比例 |
| 安全区域 | 内容区距顶部/底部各留 40px，距左右各留 24px |

### 适配规则

```
移动端：width=device-width, initial-scale=1.0
PC端：外层容器 max-width:480px, margin:0 auto, min-height:100vh
字体基准：html { font-size: calc(100vw / 375 * 16) }（移动端）
PC端固定：html { font-size: 16px }
```

## 二、全局UI规范

### 色彩体系

| 用途 | 色值 | 说明 |
|------|------|------|
| 主色 | #4A90D9 | 医疗蓝，用于标题、按钮、强调 |
| 辅色 | #5BB8F5 | 浅蓝，用于背景渐变、卡片 |
| 底色 | #F0F6FC | 极浅蓝灰，页面背景 |
| 白色 | #FFFFFF | 卡片、弹窗背景 |
| 正确/通过 | #4CAF50 | 绿色，正确反馈 |
| 警告 | #FF9800 | 橙色，顺序错误提示 |
| 错误/危险 | #F44336 | 红色，扣分、失败、致命错误 |
| 正文文字 | #333333 | 深灰 |
| 辅助文字 | #999999 | 浅灰 |

### 字体规范

| 层级 | 字号 | 字重 | 用途 |
|------|------|------|------|
| 大标题 | 24px | Bold | 游戏Logo、通关/失败大字 |
| 关卡标题 | 18px | Bold | "Level 1：隐患侦探" |
| 正文 | 14px | Regular | 旁白文字、说明文字 |
| 辅助文字 | 12px | Regular | 提示、时间戳、得分明细 |
| 按钮文字 | 16px | Medium | 所有按钮 |

字体栈：`"PingFang SC", "Microsoft YaHei", "Helvetica Neue", sans-serif`

### 通用组件

#### 顶部状态栏（游戏内页面通用）

```
┌─────────────────────────────────┐
│ [关卡标题]        [得分] [倒计时]│  高度：48px
└─────────────────────────────────┘
```
- 高度：48px，背景半透明白 rgba(255,255,255,0.9)
- 左侧：关卡标题（18px Bold，#4A90D9）
- 右侧：得分图标+数字、倒计时图标+数字（有倒计时的关卡）

#### 主按钮

- 尺寸：280 x 48px，圆角 24px
- 背景：#4A90D9，文字白色 16px Medium
- 按下态：透明度 0.8
- 禁用态：背景 #CCCCCC

#### 次按钮

- 尺寸：280 x 48px，圆角 24px
- 背景：透明，边框 1px #4A90D9，文字 #4A90D9

#### 弹窗

- 宽度：327px（页面宽度 - 48px边距）
- 背景：#FFFFFF，圆角 16px
- 遮罩层：rgba(0,0,0,0.5)
- 内边距：24px

## 五、素材规格汇总

### 输出格式

| 类型 | 格式 | 说明 |
|------|------|------|
| 插画 | PNG（透明底）或 JPG | 有背景的用JPG，需透明底的用PNG |
| 角色 | PNG（透明底） | 必须透明底 |
| 图标 | SVG 优先，PNG备选 | SVG可缩放，减小体积 |
| 证书边框 | PNG（透明底） | 中心镂空，程序填充内容 |
| 背景 | JPG | 不需要透明度，JPG体积更小 |
| Logo | PNG（透明底） | |

### 尺寸说明

- 所有标注尺寸为 @2x（即Retina屏实际像素），开发时 CSS 使用 1/2 尺寸
- 例：标注 654x800 的插画，CSS中设为 327x400，图片文件实际 654x800px
- 如需 @3x 适配高端设备，按 1.5 倍放大 @2x 尺寸

### 文件体积控制

| 类型 | 单张建议上限 | 说明 |
|------|-------------|------|
| 插画 | 200KB | JPG质量80%，或PNG用TinyPNG压缩 |
| 角色立绘 | 150KB | PNG |
| 图标 | 5KB | SVG；PNG不超过10KB |
| 背景 | 100KB | JPG质量75% |
| 证书边框 | 100KB | PNG |
| **全部素材总计** | **< 2MB** | 确保移动端首屏3秒内加载 |

## 六、技术实现方案

| 项目 | 方案 |
|------|------|
| 框架 | 原生 HTML5 + CSS3 + JavaScript（单文件或少量文件，轻量部署） |
| 页面切换 | 单页应用（SPA），所有页面为同一HTML内的不同 `<section>`，通过 `display` 切换 |
| 拖拽交互 | Touch Events（移动端） + Mouse Events（PC端），或使用轻量拖拽库 |
| 倒计时 | `setInterval` 1秒间隔，`requestAnimationFrame` 驱动动画 |
| 数据存储 | `localStorage` 存储得分记录JSON |
| 证书导出 | `html2canvas` 库将证书DOM截图为PNG供下载 |
| 部署 | 静态文件，可直接放在任意Web服务器或CDN |
