# 血管卫士：生死时速 - 游戏页面及操作流程

## 评分体系

- **初始分数**：100分
- **计分方式**：扣分制，答错扣分，最低扣至0分
- **全局规则**：任意时刻得分降至0分，立即触发"游戏失败"页面，确认后返回主页
- **得分记录**：每次游戏（无论失败或通关）均保存一条本地记录，包含玩家姓名、最终得分、评级、日期

### 各关卡扣分规则

| 关卡 | 扣分项 | 扣分 | 过关条件 |
|------|--------|------|----------|
| Level 1 | 点击干扰项 | -5分/次 | 30秒内找到3处隐患且得分>0 |
| Level 1 | 隐患原因选项：选错或漏选 | -5分/项 | |
| Level 2 | 选A或C（非最优） | -5分/次 | 选中正确选项B且得分>0 |
| Level 2 | 选D（致命错误） | 得分归0 | |
| Level 3 | 选错治疗项 | -5分/次 | 限定时间内完成正确排列 |
| Level 3 | 排列顺序错误 | -5分/次（不叠加） | |
| Level 3 | 时间到未完成 | 得分归0 | |

### 评级与证书

| 评级 | 分数区间 | 称号 | 证书 |
|------|----------|------|------|
| S级 | 90-100分 | 血栓终结者 | 金证书 |
| A级 | 70-89分 | 血管守护者 | 银证书 |
| B级 | 60-69分 | 实习新秀 | 铜证书 |
| C级 | 1-59分 | 继续修炼 | 无证书 |

### 扣分记录功能说明

#### 1. 运行时扣分日志（deductionLog）

游戏在 P2 页面初始化时创建一个扣分日志数组 `deductionLog`，在游戏全程中实时追加记录。每次发生扣分事件时，向数组中写入一条记录，结构如下：

```json
{
  "level": "Level 1",
  "action": "点击干扰项：水杯",
  "deduction": -5,
  "scoreBefore": 100,
  "scoreAfter": 95,
  "timestamp": "00:12"
}
```

#### 2. 扣分事件写入时机

| 关卡 | 写入时机 | action 示例 |
|------|----------|-------------|
| Level 1 | 点击干扰项，提示消失后 | "点击干扰项：水杯" |
| Level 1 | 隐患选项弹窗提交判定后，每个错选/漏选各写一条 | "隐患A-选错：B.会导致下肢肌肉萎缩无力" |
| Level 1 | 隐患选项弹窗提交判定后，每个漏选各写一条 | "隐患A-漏选：A.压迫腘静脉，阻碍下肢血液回流" |
| Level 1 | 30秒超时未找齐 | "超时未完成，得分归0" |
| Level 2 | 选择非正确选项后 | "选择A：立即行静脉造影" |
| Level 2 | 选择D后 | "选择D：让病人下地活动，得分归0" |
| Level 3 | 拖入干扰项后 | "拖入干扰项：让患者下地行走" |
| Level 3 | 拖入正确项但顺序错误后 | "顺序错误：将"吸氧"放在第①位（正确应为第②位）" |
| Level 3 | 30秒超时未完成 | "超时未完成排列，得分归0" |

#### 3. 得分归0的特殊处理

当扣分导致得分≤0时：
- `scoreAfter` 记录为 0（不出现负数）
- 写入一条特殊标记记录：`"action": "得分归零，游戏失败"`, `"deduction": 0`
- 该条记录写入完成后，立即触发 PF 页面

#### 4. 各页面对扣分日志的使用

| 页面 | 使用方式 |
|------|----------|
| P3 场景页（顶部得分） | 读取最新 `scoreAfter` 实时显示当前得分 |
| P4 Level 1 结算页 | 筛选 `level === "Level 1"` 的记录，按时间顺序列出扣分明细表 |
| P8 Level 2 结算页 | 筛选 `level === "Level 2"` 的记录，列出扣分明细表 |
| P11 Level 3 结算页 | 筛选 `level === "Level 3"` 的记录，列出扣分明细表 |
| PF 游戏失败页 | 显示最后一条扣分记录作为"失败原因"，并显示该关卡全部扣分明细 |
| P12 通关结算页 | 按关卡分组汇总：各关卡扣分总计 = 该关卡所有记录的 `deduction` 之和 |

#### 5. 结算页扣分明细展示格式

以 Level 1 结算页为例，扣分明细区域展示如下：

```

```

若本关无扣分，则显示：

```
┌─────────────────────────────────────────────┐
│  本关扣分明细                                │
│─────────────────────────────────────────────│
│  ★ 完美通关！本关无扣分                      │

└─────────────────────────────────────────────┘
```

#### 6. 扣分日志的生命周期

| 阶段 | 操作 |
|------|------|
| P2 点击【准备就绪】 | 创建空数组 `deductionLog = []`，设 `score = 100` |
| 游戏进行中 | 每次扣分事件实时追加记录 |
| 游戏结束（通关P12 / 失败PF） | 将 `deductionLog` 附加到得分记录中一并保存到 localStorage |
| 返回主页 / 重新挑战 | 清空当前 `deductionLog`，下局重新创建 |

## 页面流程总览

```
P1 主页 → P2 角色介绍与姓名输入页
→ P3 Level1场景页（插画+旁白+点击交互） → P3a/P3b/P3c 隐患选项弹窗
→ P4 Level1结算页
→ P5 Level2场景页（插画+旁白+选择题） → P6 正确反馈页 → P7 Level2结算页
→ P8 Level3场景页（插画+旁白+排序交互） → P9 Level3结算页
→ P10 通关结算页 → P11 证书页

任意时刻得分=0 → PF 游戏失败页 → P1

P1 主页可进入 → PR 得分记录页
```

## 详细页面说明

### P1 - 主页

- **页面内容**：
  - 游戏Logo"血管卫士：生死时速"
  - Q版医生角色立绘
  - 背景：蓝白色调医院走廊
  - 【开始游戏】按钮
  - 【得分记录】按钮
- **操作**：
  - 点击【开始游戏】→ 进入 P2
  - 点击【得分记录】→ 进入 PR

---

### PR - 得分记录页

- **页面内容**：
  - 标题："挑战记录"
  - 记录列表（按时间倒序），每条记录显示：
    - 玩家姓名
    - 最终得分
    - 评级（S/A/B/C/失败）
    - 游戏日期时间
    - 证书标记（金/银/铜/无）
  - 点击有证书的记录可查看对应证书
  - 【返回主页】按钮
- **数据存储**：使用浏览器 localStorage 存储记录数组
- **操作**：点击【返回主页】→ 返回 P1

---

### PF - 游戏失败页（全局）

- **触发条件**：任意时刻得分降至0分
- **页面内容**：
  - 画面变暗，红色警示效果
  - 大字："游戏失败"
  - 显示失败时所在关卡
  - 文字："你的得分已降至0分，挑战失败。"
  - 失败原因提示（根据扣分原因动态显示）
  - 【确认】按钮
- **数据处理**：保存本次记录（姓名、得分0、评级"失败"、日期）到本地
- **操作**：点击【确认】→ 返回 P1

---

### P2 - 角色介绍与姓名输入页

- **页面内容**：
  - 角色头像：Q版年轻医生形象
  - 文字："你是骨科新入职的住院医师。今天你将负责照护新入院的张大爷，68岁，因右侧下肢肿胀、疼痛入院，需接受进一步检查和治疗。请时刻警惕深静脉血栓（DVT）的风险！现在请告诉我你的姓名："
  - 姓名输入框（限制1-10个字符）
  - 当前得分显示：100分
  - 【准备就绪】按钮（输入为空时置灰不可点击）
- **操作**：输入姓名，点击【准备就绪】→ 记录玩家姓名，初始化得分100分，进入 P3

---

### P3 - Level 1：隐患侦探（场景页）

- **页面内容**：
  - 顶部：关卡标题"Level 1：隐患侦探"
  - 顶部右侧：当前得分 + 30秒倒计时
  - 插画：一张骨科病房全景插画，画面中包含3处隐患和若干正常物品
  - 旁白文字："张大爷因右腿肿痛刚入院不久，你来到他的病房查房。眼前的场景让你警觉——请在30秒内点击画面中不正确的行为（共3处）。"
  - 进度指示：已发现 0/3
  - 画面中可点击的区域：
    - **隐患A**：右腿（患肢）腘窝下垫着枕头，右腿自然伸直
    - **隐患B**：患者左腿盘坐在床上，长时间保持同一姿势未活动
    - **隐患C**：家属在用力揉患者的右腿（患肢）
    - **干扰项**：床头柜上的水杯、正常摆放的输液架、窗户、床栏、呼叫器等

- **操作规则**：
  - 点击正确隐患 → 该隐患高亮标记，弹出对应选项弹窗（P3a/P3b/P3c），倒计时暂停
  - 点击干扰项 → 提示"这里没有问题哦~"（1.5秒消失），**扣5分**
  - 30秒倒计时结束时未找齐3处 → **得分归0，触发 PF**
  - 3处隐患全部找到且选项弹窗全部完成且得分>0 → 进入 P4

### P3a - 隐患A选项弹窗（枕头垫在腘窝下）

- **触发条件**：在P3中点击"枕头垫在腘窝下"
- **弹窗内容**：
  - 标题："发现隐患！请选择该行为危险的原因（多选）"
  - 5个复选框选项：
    - [x] A. 压迫腘静脉，阻碍下肢血液回流 **（正确）**
    - [ ] B. 会导致下肢肌肉萎缩无力 **（错误）**
    - [x] C. 增加下肢深静脉血栓形成风险 **（正确）**
    - [ ] D. 会直接引起皮肤压疮感染 **（错误）**
    - [x] E. 导致静脉血流淤滞，符合Virchow三联征 **（正确）**
  - 【提交】按钮

- **判定规则**：
  - 正确答案为 A、C、E
  - 每选错一个（选了B或D）：扣5分
  - 每漏选一个（未选A/C/E中的某项）：扣5分
  - 全部正确：不扣分
  - 最多可扣：25分（漏选3个+选错2个）
- **提交后**：显示正确答案及解释："枕头垫在腘窝下会压迫腘静脉，导致血液回流受阻，增加DVT风险。正确做法是将枕头垫在小腿下方，使膝关节自然微屈。"
- **操作**：点击【知道了】→ 关闭弹窗，恢复倒计时，返回P3继续查找。若得分=0则触发PF

  - 上方：排列区域（5个空位，标注①②③④⑤）
  - 下方：8个可拖拽的治疗图标（乱序排列，5正确 + 3干扰）

#### 正确项及正确顺序

| 顺序 | 治疗措施 | 操作说明 |
|------|----------|----------|

| ⑤ | CTPA检查 | CT肺动脉造影，确诊PE |

#### 干扰项

| 干扰项 | 拖入反馈 |
|--------|----------|

| 按摩胸部 | "无效操作！PE不需要胸外按压！" |

#### 操作规则

- **拖入正确项到正确位置** → 图标吸附，显示绿色对勾，监护仪数值改善
- **拖入正确项到错误位置（顺序错误）** → 图标弹回，显示橙色提示"顺序不对！"，**扣5分**
- **拖入干扰项** → 图标弹回，显示红色警告提示，**扣5分**
- **扣分不叠加**：同一次拖拽操作只扣一次5分（即使同时满足多个扣分条件）
- **倒计时结束未完成排列** → **得分归0，触发 PF**
- **5个正确项按顺序排列完成** → 倒计时停止，通关
